## Overview

Design tokens become useful in Tailwind when they are registered as theme values. This enables utility classes like `bg-primary`, `p-card-padding`, and `text-lg` to resolve to governed token values rather than Tailwind's defaults.

Two integration patterns exist depending on Tailwind version:

| Version | Approach | Where tokens live |
|---|---|---|
| **Tailwind v4** | `@theme` directive in CSS | `app.css` or a dedicated `tokens.css` |
| **Tailwind v3** | `theme.extend` in config | `tailwind.config.ts` |

Both approaches consume the same CSS custom properties generated by Style Dictionary.

## Tailwind v4: The @theme Directive

Tailwind v4 introduces `@theme` as a CSS-native way to define theme values. Tokens map directly to CSS custom properties inside `@theme` blocks.

### Basic setup: `app.css`

```css
@import "tailwindcss";

/* Import token CSS generated by Style Dictionary */
@import "./tokens.css";

/*
 * Register tokens as Tailwind theme values.
 * This makes utilities like bg-primary, text-surface, p-card-padding available.
 */
@theme {
  /* Colors -- semantic tier */
  --color-primary: var(--color-primary);
  --color-primary-hover: var(--color-primary-hover);
  --color-on-primary: var(--color-on-primary);
  --color-secondary: var(--color-secondary);
  --color-surface: var(--color-surface);
  --color-surface-raised: var(--color-surface-raised);
  --color-surface-sunken: var(--color-surface-sunken);
  --color-background: var(--color-background);
  --color-border: var(--color-border);
  --color-border-strong: var(--color-border-strong);
  --color-text: var(--color-text);
  --color-text-muted: var(--color-text-muted);
  --color-text-subtle: var(--color-text-subtle);
  --color-success: var(--color-success);
  --color-error: var(--color-error);

  /* Spacing -- semantic tier */
  --spacing-card-padding: var(--spacing-card-padding);
  --spacing-section-gap: var(--spacing-section-gap);
  --spacing-input-x: var(--spacing-input-x);
  --spacing-input-y: var(--spacing-input-y);
  --spacing-stack-sm: var(--spacing-stack-sm);
  --spacing-stack-md: var(--spacing-stack-md);
  --spacing-stack-lg: var(--spacing-stack-lg);

  /* Border radius -- semantic tier */
  --radius-card: var(--radius-card);
  --radius-button: var(--radius-button);
  --radius-input: var(--radius-input);
  --radius-badge: var(--radius-badge);

  /* Shadows -- semantic tier */
  --shadow-card: var(--shadow-card);
  --shadow-dropdown: var(--shadow-dropdown);
  --shadow-dialog: var(--shadow-dialog);

  /* Font sizes */
  --text-xs: var(--font-size-xs);
  --text-sm: var(--font-size-sm);
  --text-base: var(--font-size-base);
  --text-lg: var(--font-size-lg);
  --text-xl: var(--font-size-xl);
  --text-2xl: var(--font-size-2xl);
  --text-3xl: var(--font-size-3xl);
  --text-4xl: var(--font-size-4xl);

  /* Transitions */
  --transition-timing-fast: var(--motion-duration-fast);
  --transition-timing-normal: var(--motion-duration-normal);
  --transition-timing-slow: var(--motion-duration-slow);
}
```

### Resulting utility classes

After `@theme` registration, these classes work out of the box:

```html
<div class="bg-surface text-text p-card-padding rounded-card shadow-card">
  <h2 class="text-2xl font-semibold text-text">Card Title</h2>
  <p class="text-sm text-text-muted mt-stack-sm">Card content here.</p>
  <button class="bg-primary text-on-primary px-4 py-2 rounded-button hover:bg-primary-hover">
    Action
  </button>
</div>
```

### Overriding Tailwind defaults

To replace Tailwind's built-in color palette entirely with your tokens, use `@theme` without any fallback:

```css
@theme {
  /* Remove all default colors, use only token colors */
  --color-*: initial;

  /* Then define your token colors */
  --color-primary: var(--color-primary);
  --color-surface: var(--color-surface);
  /* ... */
}
```

## Tailwind v3: theme.extend Configuration

In Tailwind v3, tokens are wired through `tailwind.config.ts` by referencing CSS custom properties.

### Configuration: `tailwind.config.ts`

```typescript
import type { Config } from 'tailwindcss';

const config: Config = {
  content: ['./src/**/*.{ts,tsx,html}'],
  darkMode: ['selector', '[data-theme="dark"]'],
  theme: {
    extend: {
      colors: {
        primary: {
          DEFAULT: 'var(--color-primary)',
          hover: 'var(--color-primary-hover)',
        },
        'on-primary': 'var(--color-on-primary)',
        secondary: 'var(--color-secondary)',
        surface: {
          DEFAULT: 'var(--color-surface)',
          raised: 'var(--color-surface-raised)',
          sunken: 'var(--color-surface-sunken)',
        },
        background: 'var(--color-background)',
        border: {
          DEFAULT: 'var(--color-border)',
          strong: 'var(--color-border-strong)',
        },
        text: {
          DEFAULT: 'var(--color-text)',
          muted: 'var(--color-text-muted)',
          subtle: 'var(--color-text-subtle)',
        },
        success: 'var(--color-success)',
        error: 'var(--color-error)',
      },
      spacing: {
        'card-padding': 'var(--spacing-card-padding)',
        'section-gap': 'var(--spacing-section-gap)',

      boxShadow: {

      fontSize: {
        xs: 'var(--font-size-xs)',
```

### Including token CSS

Load the Style Dictionary output in your global stylesheet before Tailwind layers:

```css
/* globals.css */
@import './tokens.css';

@tailwind base;
@tailwind components;
@tailwind utilities;
```

## Consuming Tokens in Components

### React + Tailwind (shadcn/radix-ui pattern)

```tsx
import { cn } from '@/lib/utils';

interface ButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {
  variant?: 'primary' | 'secondary' | 'ghost';
  size?: 'sm' | 'md' | 'lg';
}

const variants = {
  primary: 'bg-primary text-on-primary hover:bg-primary-hover shadow-sm',
  secondary: 'bg-surface text-text border border-border hover:bg-surface-sunken',
  ghost: 'bg-transparent text-text-muted hover:bg-surface-sunken hover:text-text',
};

const sizes = {
  sm: 'px-3 py-1.5 text-xs rounded-button',
  md: 'px-4 py-2 text-sm rounded-button',
  lg: 'px-6 py-3 text-base rounded-button',
};

export function Button({
  variant = 'primary',
  size = 'md',
  className,
  ...props
}: ButtonProps) {
  return (
    <button
      className={cn(
        'inline-flex items-center justify-center font-semibold',
        'transition-colors duration-fast',
        'focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-primary focus-visible:ring-offset-2',
        'disabled:pointer-events-none disabled:opacity-50',
        variants[variant],
        sizes[size],
        className
      )}
      {...props}
    />
  );
}
```

### Direct CSS custom property access

When Tailwind utilities are insufficient, access tokens directly:

```css
.custom-gradient-card {
  background: linear-gradient(
    135deg,
    var(--color-surface) 0%,

  box-shadow: var(--shadow-card);
  transition: box-shadow var(--motion-duration-normal) var(--motion-easing-ease-out);
}

.custom-gradient-card:hover {
  box-shadow: var(--shadow-dropdown);
}
```

## Dark Mode with Token Swapping

Dark mode works by swapping semantic token values. Primitives remain unchanged.

### Token override layer: `tokens-dark.css`

```css
[data-theme="dark"] {
  /* Swap semantic aliases -- primitives stay the same */
  --color-primary: var(--color-blue-400);
  --color-primary-hover: var(--color-blue-300);
  --color-on-primary: var(--color-neutral-950);

  --color-surface: var(--color-neutral-900);
  --color-surface-raised: var(--color-neutral-800);
  --color-surface-sunken: var(--color-neutral-950);
  --color-background: var(--color-neutral-950);

  --color-border: var(--color-neutral-700);
  --color-border-strong: var(--color-neutral-600);

  --color-text: var(--color-neutral-50);
  --color-text-muted: var(--color-neutral-400);
  --color-text-subtle: var(--color-neutral-500);

  --color-success: var(--color-green-500);
  --color-error: var(--color-red-500);

  /* Shadows get subtler in dark mode */
  --shadow-card: 0 4px 6px -1px oklch(0 0 0 / 0.3);
  --shadow-dropdown: 0 10px 15px -3px oklch(0 0 0 / 0.4);
  --shadow-dialog: 0 20px 25px -5px oklch(0 0 0 / 0.5);
}
```

### Theme switching

```typescript
function setTheme(theme: 'light' | 'dark' | 'system') {
  if (theme === 'system') {
    const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
    document.documentElement.setAttribute('data-theme', prefersDark ? 'dark' : 'light');
  } else {
    document.documentElement.setAttribute('data-theme', theme);
  }
  localStorage.setItem('theme', theme);
}

// Initialize on page load
const stored = localStorage.getItem('theme') as 'light' | 'dark' | 'system' | null;
setTheme(stored ?? 'system');

// Listen for system changes when in system mode
window.matchMedia('(prefers-color-scheme: dark)').addEventListener('change', () => {
  if (localStorage.getItem('theme') === 'system') {
    setTheme('system');
  }
});
```

### Tailwind v3 dark mode selector

With `darkMode: ['selector', '[data-theme="dark"]']` in your Tailwind config, the `dark:` variant activates when `data-theme="dark"` is on the root element. This works automatically with the token swap approach -- no `dark:` prefixes needed for token-based colors since the custom properties themselves change.

Use `dark:` only for structural changes:

```html
<!-- Token colors swap automatically via CSS custom properties -->
<div class="bg-surface text-text p-card-padding rounded-card shadow-card">
  <!-- Use dark: only for layout or structural differences -->
  <div class="grid grid-cols-1 dark:grid-cols-2">
    ...
  </div>
</div>
```

## shadcn/ui Integration

shadcn/ui uses CSS custom properties for theming. Map your design tokens to shadcn's expected variable names, or reconfigure shadcn to use your token names.

### Option A: Map tokens to shadcn variable names

```css
:root {
  /* shadcn expects these variable names */
  --background: var(--color-background);
  --foreground: var(--color-text);
  --card: var(--color-surface);
  --card-foreground: var(--color-text);
  --primary: var(--color-primary);
  --primary-foreground: var(--color-on-primary);
  --secondary: var(--color-secondary);
  --secondary-foreground: var(--color-text);
  --muted: var(--color-surface-sunken);
  --muted-foreground: var(--color-text-muted);
  --accent: var(--color-surface-sunken);
  --accent-foreground: var(--color-text);
  --destructive: var(--color-error);
  --border: var(--color-border);
  --input: var(--color-border);
  --ring: var(--color-primary);
  --radius: var(--radius-md);
}
```

### Option B: Reconfigure shadcn components to use token names

When adding shadcn components, update their class references to use your token utilities directly. This is more work up front but avoids the indirection of mapping layers.

## Automated Token-to-Tailwind Pipeline

For projects using Style Dictionary, generate the Tailwind config automatically:

### Style Dictionary format for Tailwind v3

```javascript
// In style-dictionary.config.mjs
StyleDictionary.registerFormat({
  name: 'tailwind/theme-extend',
  format: ({ dictionary }) => {
    const grouped = {};

    dictionary.allTokens.forEach((token) => {
      const [category, ...rest] = token.path;
      if (!grouped[category]) grouped[category] = {};

      const key = rest.join('-');
      grouped[category][key] = `var(--${token.name})`;
    });

    return `// Auto-generated by Style Dictionary -- do not edit
export const tokens = ${JSON.stringify(grouped, null, 2)};
`;
  },
});
```

Then import in your Tailwind config:

```typescript
// tailwind.config.ts
import { tokens } from './build/tailwind/tokens.js';

export default {
  theme: {
    extend: {
      colors: tokens.color,
      spacing: tokens.spacing,
      borderRadius: tokens.radius,
      boxShadow: tokens.shadow,
      fontSize: tokens.font?.size,
    },
  },
};
```

This ensures the Tailwind config stays in sync with tokens without manual updates.
