NEW = phases added in this milestone   EXISTING = original phases

Phase 0: Fingerprint Check NEW

Hash source files with SHA-256 and compare against .cover/fingerprints.json. Unchanged files with a previous pass result are skipped.

Inputs: source files in scope, .cover/fingerprints.json

Outputs: reduced scope (changed/new/failing files only)

Flags: --no-cache bypasses this phase entirely

Ref: references/fingerprint-gating.md

Phase 1: Discover EXISTING

Scan for test frameworks (Vitest, Jest, pytest, Playwright) and identify untested source files, exports, and UI components.

Inputs: project root, scope argument

Outputs: detected frameworks, untested files list, tier capabilities

Flags: --tier limits which frameworks are relevant

Phase 2: Analyze EXISTING

Run existing tests and collect baseline coverage (lines, branches, functions). Scaffold a test framework if none is detected.

Inputs: detected frameworks from Phase 1

Outputs: baseline coverage metrics, pass/fail counts, existing test paths

Flags: --tier controls which runners execute

Phase 2b: Saved Flow Lookup NEW

When --flow <name> is provided, look up .cover/flows/<name>.yaml. If found, skip Phases 2 and 3 and proceed directly to Phase 4 with the saved flow steps.

Inputs: --flow flag value, .cover/flows/ directory

Outputs: flow YAML steps (or error with available flows list)

Flags: --flow <name> activates this phase

Ref: references/saved-flows.md

Phase 3: Generate EXISTING

Generate test files per tier using parallel agents. Effort level controls tier selection and agent count. E2E tier uses agent-browser exploration followed by Playwright codification.

Inputs: analysis results, scope, effort level

Outputs: generated test files

Flags: --tier overrides effort defaults

Phase 3b: ARIA Baseline Capture NEW

Active when --tier=e2e and --no-aria is not set. Navigate to each page under test, wait for network idle, and capture the ARIA accessibility tree as a JSON baseline in .cover/aria-baselines/.

Inputs: generated E2E test files, running application

Outputs: ARIA baseline JSON files

Flags: --no-aria skips this phase

Ref: references/aria-diffing.md

Phase 4: Execute EXISTING

Run all generated tests. Capture exit codes, pass/fail per test, error messages, and stack traces.

Inputs: generated test files

Outputs: test results (pass/fail, errors, stack traces)

Phase 4b: ARIA Diff NEW

Re-capture ARIA trees after test execution and diff node-by-node against baselines. Classify changes as CRITICAL, WARNING, or INFO.

Inputs: ARIA baselines from Phase 3b, current page state

Outputs: severity-classified ARIA diff results

Flags: --no-aria skips this phase

Ref: references/aria-diffing.md

Phase 5: Heal EXISTING

Fix failing tests without modifying source code. Up to 3 iterations: (1) obvious fixes, (2) interaction errors, (3) edge cases. Source bugs are reported but not fixed.

Inputs: failing test results from Phase 4

Outputs: healed test files, source bug reports

Ref: references/heal-loop.md

Phase 6: Report EXISTING

Generate coverage comparison (before/after), test summary, source bugs, fingerprint cache stats, and ARIA diff results.

Inputs: all prior phase outputs

Outputs: formatted coverage report with delta table

Flags: --status-protocol appends machine-parseable summary

Ref: references/coverage-report.md, references/status-protocol.md

Decision Trees

Phase 0: Fingerprint Gating

Does .cover/fingerprints.json exist?
→ No: first run, all files proceed to Phase 1
→ Yes: hash each source file with SHA-256
Hash matches + last result was pass?
→ Yes: skip file (cached)
→ No: file proceeds to Phase 1
--no-cache flag set?
→ Yes: bypass all caching, test everything

Phase 2b: Saved Flow Lookup

Is --flow <name> flag provided?
→ No: proceed to Phase 2 (Analyze) + Phase 3 (Generate) normally
→ Yes: look up .cover/flows/<name>.yaml
Flow file found?
→ Yes: skip Phase 2+3, jump to Phase 4 with saved steps
→ No: error with list of available flows

Phase 3b: ARIA Baseline

Is --tier=e2e requested?
→ No: skip ARIA phases entirely (3b + 4b)
→ Yes: check ARIA flag
Is --no-aria set?
→ Yes: skip ARIA baseline capture
→ No: capture ARIA tree for each page under test

Phase 5: Heal Loop

All tests pass?
→ Yes: proceed to Phase 6 (Report)
→ No: enter heal loop
Iteration < max (3 for high effort)?
→ Yes: classify failure, apply fix, re-run
→ No: report remaining failures in Phase 6
Failure is a source bug?
→ Yes: report bug, do NOT fix source code

New Features in This Milestone

Saved Flows

Reusable YAML definitions for E2E user journeys. Skip AI generation for known paths -- version-control critical flows alongside application code.

Supports selector fallback chains, variable substitution, conditional steps, and flow composition via the setup field.

--flow <name>

references/saved-flows.md

ARIA Diffing

Capture ARIA accessibility trees before and after test execution, then diff node-by-node. Catches regressions invisible to visual and unit tests: lost roles, missing labels, broken landmarks.

Changes classified as CRITICAL, WARNING, or INFO with a decision tree for distinguishing regressions from intentional changes.

--no-aria (to disable)

references/aria-diffing.md

Fingerprint Gating

SHA-256 hash comparison against .cover/fingerprints.json from the previous run. Unchanged files with passing results are skipped, making incremental runs practical on large codebases.

First run processes everything. Subsequent runs only test changed, new, or previously failing files.

--no-cache (to bypass)

references/fingerprint-gating.md

Status Protocol

Machine-parseable [COVER_*] event lines interleaved with human-readable output. Filter lines starting with [COVER_ to extract the structured event stream.

Designed for CI pipelines, monitoring dashboards, structured logging (ELK, Datadog), and automated quality gates.

--status-protocol

references/status-protocol.md

Getting Started

Basic Commands

CommandWhat It Does
/cover src/auth/Generate tests for all files in src/auth/ (auto-detect tiers + effort)
/cover --tier=unit src/utils/Unit tests only for src/utils/
/cover --tier=e2e checkout-flowE2E tests for checkout flow using Playwright + agent-browser
/cover --tier=e2e --flow loginReplay saved login flow (skip AI generation, save tokens)

Advanced Commands

CommandWhat It Does
/cover --no-cache src/Force full re-test (ignore fingerprint cache)
/cover --tier=e2e --no-ariaE2E tests without ARIA accessibility diffing
/cover --status-protocol src/Emit [COVER_*] machine-parseable lines for CI parsing
/cover --tier=e2e --flow login --status-protocolCombine: saved flow + status protocol

Setting Up Saved Flows

# 1. Create the flows directory
mkdir -p .cover/flows

# 2. Create a flow file (example: login)
# .cover/flows/login.yaml
name: login
description: Standard email+password authentication
url: ${BASE_URL}/login
env: [TEST_EMAIL, TEST_PASSWORD]
steps:
  - action: navigate
    url: ${BASE_URL}/login
  - action: fill
    selector:
      primary: "[data-testid='email-input']"
      fallback_role: "textbox[name='Email']"
    value: ${TEST_EMAIL}
  - action: fill
    selector:
      primary: "[data-testid='password-input']"
      fallback_label: "Password"
    value: ${TEST_PASSWORD}
  - action: click
    selector:
      primary: "[data-testid='submit-btn']"
      fallback_role: "button[name='Sign In']"
  - action: assert
    type: url
    expected: "**/dashboard"
    timeout: 5000

# 3. Run with the flow
/cover --tier=e2e --flow login

How to Validate the Pipeline

The /cover skill is documentation-only (SKILL.md + reference docs). Validation means verifying Claude follows the instructions correctly on a real project.

Level 1: Smoke Test 5 min

# In any project with source code:
/cover --tier=unit src/

# Verify Claude:
  [x] Runs Phase 0 (fingerprint check) before discovering
  [x] Generates and runs unit tests
  [x] Produces coverage comparison (before/after)
  [x] Updates .cover/fingerprints.json after success

Level 2: Feature Validation 30 min

FeatureTestExpected
Fingerprint Gating Run /cover twice without changes Second run: "N files cached", skips unchanged
--no-cache Run /cover --no-cache after cached run All files tested regardless of cache
Saved Flows Create flow YAML, run /cover --flow test --tier=e2e Skips Phase 2+3, uses flow steps
Flow not found /cover --flow nonexistent --tier=e2e Error listing available flows
ARIA Diffing /cover --tier=e2e with running app Captures ARIA baseline (3b) and diffs (4b)
--no-aria /cover --tier=e2e --no-aria Phases 3b and 4b skipped
Status Protocol /cover --status-protocol src/ [COVER_STEP_*] lines in output
Heal Loop Run on code with known bug Test fails, heals test, reports source bug

Level 3: Full E2E Pipeline 1 hour

# Prerequisites:
  - Web app running at http://localhost:3000
  - agent-browser CLI installed
  - .cover/flows/login.yaml created

# Full command:
/cover --tier=e2e --flow login --status-protocol

# Verify complete pipeline:
  [x] Phase 0: Fingerprint check runs first
  [x] Phase 2b: Saved flow loaded (skips 2+3)
  [x] Phase 3b: ARIA baseline captured
  [x] Phase 4: E2E tests run via Playwright
  [x] Phase 4b: ARIA diff against baseline
  [x] Phase 5: Heal loop if failures
  [x] Phase 6: Report with coverage + ARIA + cache
  [x] Status protocol lines present
  [x] .cover/fingerprints.json updated
  [x] .cover/aria-baselines/ populated
